    • GIT IGNORE

    PASO 1

    Si aún no tienes un archivo .gitignore, créalo en el directorio raíz de tu repositorio. Puedes hacerlo con el comando: touch .gitignore Luego, abre el archivo en un editor de texto y añade los patrones de archivos o directorios que deseas ignorar. Por ejemplo: # Ignorar archivos temporales *.log # Ignorar el directorio de dependencias node_modules/ # Ignorar archivos de configuración

    PASO 2

    Una vez configurado el .gitignore, añade el archivo al área de preparación de Git con el comando: git add .gitignore Luego, realiza un commit para guardar los cambios en tu repositorio: git commit -m "Añadido archivo .gitignore para excluir archivos no deseados"

    PASO 3

    Si ya has añadido archivos que ahora están en tu .gitignore, elimínalos del repositorio (pero no del sistema de archivos local) para que Git deje de rastrearlos. Usa el siguiente comando: git rm --cached Luego, realiza un commit para actualizar el repositorio: git commit -m "Eliminar archivos del repositorio que ahora están en .gitignore"
